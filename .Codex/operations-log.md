## 技术选型理由
- 为什么用这个方案：在现有静态 HTML 中通过数据属性控制显示状态，可保持 Hexo 构建链路不变。
- 优势：零新增依赖、改动范围小、可渐进增强、分类筛选与加载更多可以共享状态。
- 劣势和风险：所有文章仍存在于生成的 HTML 中，主要优化首屏视觉和交互，不减少 HTML 文件体积。

## 关键风险点
- 并发问题：纯前端同步交互，无共享异步状态。
- 边界条件：筛选结果少于一批、年份分组全部隐藏、文章仅有一个标题或没有标题。
- 性能瓶颈：首页包含 139 篇文章，交互需避免反复进行昂贵 DOM 查询。
- 安全考虑：不拼接用户输入，不引入外部数据提交。

## 缺陷修复 - Markdown 公式与文章目录
时间：2026-06-06 21:18:47

- 现象：KaTeX 公式下方显示一份压平后的公式文本；部分仅有一个标题的文章不显示目录。
- 根因：
  - KaTeX CSS 使用外部 CDN，实际页面未加载该样式，导致无障碍 MathML 层可见。
  - 目录条件被改成少于两个标题时隐藏，偏离原有“有标题就显示”的行为。
- 修复：
  - 复用项目已安装的 `katex` 包，将 `katex.min.css` 和字体资源放入主题本地静态资源。
  - 头部改为引用本地 KaTeX CSS，取消外部 CDN 依赖。
  - 增加 MathML 视觉隐藏兜底，保留其无障碍语义。
  - 目录恢复为仅在完全没有标题时隐藏。
